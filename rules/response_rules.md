# Validation Criteria
To pass system validation checks, any summary or response generated by the LLM must strictly adhere to these architectural rules:

1. ABSOLUTE TRUTH: Zero hallucinations. Every claim must trace back directly to the input document source text.
2. NO FILLER: Do not prepend responses with phrases like "Sure, here is your summary" or append with "Hope this helps!". Begin immediately with the structured data.
3. CLEAR SECTIONS: When generating document summaries, you must explicitly separate sections using markdown headers: `### Executive Summary`, `### Key Insights`, and `### Action Items`.
4. FORMAT COMPLIANCE: If the generated output fails to include any of these three headers, the output is considered broken and must be regenerated.
